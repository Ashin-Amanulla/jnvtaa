# JNVTAA Alumni Website Implementation Plan

## Tech Stack

- **Frontend**: Vite + React, Tailwind CSS, Shadcn UI, Zustand (state), TanStack Query (API state)
- **Backend**: Node.js + Express, MongoDB + Mongoose, JWT auth, Joi validation
- **Features**: Progressive Web App, real-time updates, secure authentication

## Project Structure Setup

### Backend Structure

```
backend/
├── config/           # DB, environment configs
├── helpers/          # Utility functions
├── middlewares/      # Auth, error handling, validation
├── modules/
│   ├── auth/        # Login, register, verify
│   ├── users/       # Alumni profiles
│   ├── events/      # Events CRUD
│   ├── news/        # News & updates
│   ├── gallery/     # Photos/videos
│   ├── donations/   # Fundraising
│   ├── jobs/        # Job board
│   └── batches/     # Batch management
├── seeds/           # Dummy data
└── validators/      # Joi schemas
```

### Frontend Structure

```
frontend/src/
├── api/             # API client, endpoints
├── components/      # Reusable components
├── layout/          # Header, Footer, Sidebar
├── pages/           # Route pages
├── store/           # Zustand stores
├── hooks/           # Custom hooks
├── ui/              # Shadcn UI components
└── utils/           # Helper functions
```

## Core Implementation Steps

### 1. Backend Foundation

- Set up Express server with middleware (CORS, helmet, rate limiting)
- Configure MongoDB connection with Mongoose
- Implement centralized error handling middleware
- Create JWT authentication system with refresh tokens
- Set up Joi validation middleware
- Create seed data for development

### 2. Authentication & User Management

- **Auth Module**: Register, login, logout, verify email, forgot password
- **User Model**: Name, batch, email, phone, location, profession, bio, avatar, social links, privacy settings
- **Alumni Verification**: Admin approval system for new registrations
- **Profile Management**: CRUD operations, profile completion tracking

### 3. Core Modules (Backend)

- **Batches Module**: Batch info, batch-specific galleries, reunion data
- **Events Module**: CRUD, RSVP system, event categories (reunions, meets, virtual)
- **News Module**: Articles with rich text, categories, comments
- **Gallery Module**: Image/video upload with Cloudinary/AWS S3, albums, tags
- **Donations Module**: Campaign creation, payment integration (Razorpay), tracking
- **Jobs Module**: Post jobs, apply, filter by industry/location

### 4. Frontend Core Setup

- Configure Vite with path aliases, environment variables
- Set up Tailwind CSS with custom color palette (school colors)
- Install and configure Shadcn UI components
- Create layout components (Navbar, Footer, Sidebar)
- Set up React Router with protected routes
- Configure TanStack Query with default options

### 5. State Management

- **Auth Store**: User session, login state, permissions
- **UI Store**: Theme (dark/light), sidebar state, notifications
- Create API hooks with TanStack Query for all modules
- Implement optimistic updates for better UX

### 6. Key Pages (Frontend)

**Public Pages:**

- **Home**: Hero, stats counters, featured alumni, upcoming events, news preview
- **About**: School history, JNVTAA mission, leadership team
- **Directory**: Searchable alumni list with filters (batch, location, profession), interactive map
- **Batch Pages**: Batch-specific alumni, group photos, reunion info
- **Events**: Calendar view, upcoming/past events, RSVP functionality
- **News**: Articles grid, individual article pages, categories
- **Gallery**: Photo/video grid with lightbox, filter by event/batch
- **Contact**: Contact form, school address, social links

**Protected Pages:**

- **Dashboard**: Profile stats, upcoming events, notifications
- **My Profile**: Edit profile, upload photo, privacy controls
- **Networking**: Browse alumni, send connection requests, messaging
- **Job Board**: Browse/post jobs
- **Donate**: Donation portal with campaign selection

### 7. Advanced Features

- **Search**: Global search across alumni, events, news
- **Notifications**: Real-time push notifications for events, messages
- **PWA**: Service worker, offline support, installable
- **Social Features**: Like, comment, share on posts/events
- **Memory Wall**: Share stories/memories with moderation
- **Alumni Map**: Interactive world map showing alumni locations
- **Birthday Reminders**: Automated birthday wishes
- **Email System**: Automated emails for events, newsletters

### 8. UI Components to Build

- AlumniCard, EventCard, NewsCard, GalleryItem
- SearchBar with filters
- Stats counter with animation
- Timeline component
- Lightbox/Modal
- Form components (Input, Select, DatePicker, FileUpload)
- Pagination, Infinite scroll
- Loading skeletons
- Toast notifications
- Badge/Tag components

### 9. Admin Panel

- Dashboard with analytics (total alumni, events, donations)
- User management (approve/reject registrations, verify batches)
- Content moderation (gallery, memory wall, comments)
- Event management
- Email campaign creator
- Donation tracking and reports

### 10. Security & Performance

- Input validation and sanitization (frontend + backend)
- Rate limiting on auth endpoints
- Image optimization and lazy loading
- Code splitting and lazy route loading
- Redis caching for frequently accessed data
- Database indexing on search fields
- HTTPS enforcement
- CORS configuration
- XSS and CSRF protection

### 11. Testing & Quality

- Set up ESLint and Prettier
- API endpoint testing
- Responsive design testing
- Browser compatibility testing
- Performance optimization (Lighthouse scores)
- Accessibility audit (WCAG compliance)

### 12. Deployment Preparation

- Environment-specific configs (dev, staging, production)
- Docker configuration for containerization
- PM2 setup for process management
- Database backup strategy
- CDN setup for static assets
- SSL certificate setup
- Domain and hosting configuration

## Dummy Data Requirements

- 50+ alumni profiles (varied batches, locations, professions)
- 10+ batches (2000-2024)
- 15+ upcoming/past events
- 20+ news articles
- 100+ gallery items (photos/videos)
- 5+ donation campaigns
- 10+ job listings
- Sample committee members

## Design Guidelines

- **Colors**: Derive from JNV branding (school colors)
- **Typography**: Clean, readable fonts (Inter, Poppins, or similar)
- **Imagery**: Mix of nostalgic school photos and modern design
- **Animations**: Subtle transitions, smooth scrolls, hover effects
- **Mobile-first**: All components responsive from mobile up
- **Accessibility**: High contrast, keyboard navigation, screen reader support

## Priority Order

1. Auth system + User profiles
2. Alumni directory with search/filter
3. Events module with RSVP
4. News & Gallery
5. Dashboard and profile management
6. Networking features (messaging, connections)
7. Donations & Job board
8. Advanced features (PWA, notifications, admin panel)

## Key Files to Create

- `backend/config/database.js` - MongoDB connection
- `backend/config/auth.js` - JWT configuration
- `backend/middlewares/auth.middleware.js` - Auth verification
- `backend/middlewares/error.middleware.js` - Error handling
- `backend/seeds/index.js` - Seed all dummy data
- `frontend/src/api/client.js` - Axios instance
- `frontend/src/store/auth.js` - Auth Zustand store
- `frontend/src/components/layout/Navbar.jsx` - Main navigation
- `frontend/tailwind.config.js` - Custom theme configuration
